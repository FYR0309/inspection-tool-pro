# v3 统一模板导入引擎 — 设计方案

> 2026-07-12 · 基于 v2.5 完成后的下一轮迭代

---

## 一、核心理念

**输入什么格式，输出什么格式。深度克隆，保留原样式。**

| | DOCX | XLSX |
|------|------|------|
| 输入 | 客户 Word 模板 | 客户 Excel 模板 |
| 输出 | Word 报告（含照片） | Excel 报告（纯文字） |
| 样式 | 全保留（字体/颜色/边框/Logo/签名/页边距） | 全保留（字体/颜色/边框/列宽/合并单元格/下拉验证/条件格式） |
| 克隆方式 | 复制原始 XML 骨架 + 往表格行填数据 | 复制原始 XML 骨架 + 往单元格填数据 |

---

## 二、导入流程（统一入口）

```
首页「导入新模板（.docx / .xlsx）」按钮
              │
              ▼
         选择文件
              │
      ┌──────┴──────┐
      │             │
    .docx         .xlsx
      │             │
  docx-parser   xlsx-parser（新增）
      │             │
      └──────┬──────┘
             ▼
      AI 识别列结构 + 数据区域
             │
             ▼
      映射表确认面板
      ├─ 每列：列名 + 识别状态 + 类型下拉
      ├─ DOCX 类型选项：序号/描述/照片/备注/文字
      ├─ XLSX 类型选项：序号/描述/文字/日期/数字
      ├─ 数据区域：AI 自动识别 + 可手动调整
      └─ 用户确认 → 保存
             │
             ▼
      原始文件 → IndexedDB（克隆引擎用）
      模板元数据 → IndexedDB（列表展示用）
             │
             ▼
      首页「我的模板」混排 DOCX + XLSX
```

## 三、XLSX 特殊处理

### 3.1 智能 Sheet 检测

| 情况 | 行为 |
|------|------|
| 只有 1 个 Sheet | 自动使用，不提醒 |
| 多个 Sheet，仅 1 个有内容 | 自动使用，不提醒 |
| 多个 Sheet 都有内容 | 弹 Sheet 选择器，标注识别结果辅助用户判断 |

### 3.2 列类型选项（不同于 DOCX）

DOCX 有「照片」类型，XLSX 没有。XLSX 额外有「日期」「数字」类型：

- 序号 / 描述 / 文字 / 日期 / 数字

符号（✓✗○△）视为普通文字，不需要特殊类型。克隆时原始模板的下拉验证、条件格式自动继承。

### 3.3 照片策略

XLSX 不支持单元格内嵌图片，定位为纯文字检查表。需要照片的场景用 DOCX 模板。

## 四、AI 识别 + 手动调整双保险

```
AI 自动识别：
  ├─ 数据区域边界（跳过标题行/汇总行/空行空列）
  ├─ 表头行位置（含多行合并表头 → 合并为单行）
  └─ 每列类型（基于列名关键词 + 数据样本推断）

用户手动调整：
  ├─ 点击"手动调整"重新框选数据区域
  ├─ 下拉修改每列类型
  └─ 编辑模板名称
```

## 五、存储

统一 IndexedDB，现有存储结构扩展：

```
模板记录
├─ id, name, description, industry（现有）
├─ sourceFormat: 'docx' | 'xlsx'（新增）
├─ columns[]（现有，列结构）
├─ originalFile: ArrayBuffer（新增，原始文件供克隆引擎读取）
└─ titleTemplate, overviewTemplate, footerTemplate ...（现有）
```

原始文件存储与现有 `docx-template-cloner.js` 逻辑统一，XLSX 走同一套管道。

---

## 六、模板卡片

首页「我的模板」区域混排，格式标签区分：

- DOCX：蓝色标签 `[DOCX]`
- XLSX：绿色标签 `[XLSX]`
- 卡片显示：格式标签 + 模板名称 + 描述 + 编辑/删除按钮
- 不显示行业标签（如建筑/仓储/电力）——简化卡片信息

---

## 七、入口变更

- 现有「导入新模板（.docx / .json）」→ 「导入新模板（.docx / .xlsx）」
- `.json` 手动导入合并到导入面板内作为高级选项（不显眼但可用）
- `showImportPanel()` 保持为统一入口，内部根据扩展名分流

---

## 八、不涉及的部分

- 内置模板不变（safety / 5s / universal 保持 DOCX）
- 现有的报告生成流程不变（DOCX 走 docx-gen.js + docx-template-cloner.js）
- 激活/付费/使用次数逻辑不变
- XLSX 生成的报告暂不计入"已生成报告"历史（后续迭代加）
